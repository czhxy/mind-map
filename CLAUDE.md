# XMind 脑图整理规范

本文件夹中所有 XMind 脑图遵循以下两种风格之一。整理时默认使用**深度分析式**，除非用户明确指定另一种。

---

## 风格一：深度分析式（默认）

**核心特征：节点标题简洁，详细内容在 notes 中。**

适用场景：技术深度分析、底层原理讲解、需要面试准备的专题。

### 结构规范

```
根节点：主题名称
├── 1. 基础概念/背景（可选，需要铺垫时加入）
├── 2. 核心详解（按子主题拆分，如 特性/接口/寄存器/典型代表）
├── 3. 相关对比（如 vs 同类技术，多维度横向对比）
├── 4. 实践/实战（如项目中的使用、配合策略）
├── 5. 总结/收获（1-3 条关键认知）
```

板块数量可根据主题灵活调整，但**必须有"核心收获"收尾节点**。

### 节点规范

- 每个节点**必须有 notes**，notes 中写详细解释（原理、原因、代码示例等）
- 节点标题用加序号短语（如"2. Flash 详解"、"核心特性"）
- notes 中可用 `\n` 分隔多段内容
- notes 内容要有深度，不是标题的重复，而是补充**为什么**、**怎么用**、**常见坑**

## 风格二：导图式

**核心特征：详细内容直接放在节点标题中，几乎无 notes。**

适用场景：概念对比辨析、面试速查、提纲式知识梳理。

### 结构规范

```
根节点：主题名称
├── 对比组 1（如"指针数组 vs 数组指针"）
│   ├── 概念 A（标题含定义）
│   │   ├── 使用场景：xxx
│   │   ├── 关键辨析：xxx
│   │   └── sizeof差异：xxx
│   └── 概念 B（标题含定义）
│       ├── 使用场景：xxx
│       └── 注意事项：xxx
├── 对比组 2
│   └── ...
```

### 节点规范

- 节点标题直接写详细内容，使用 `\n` 换行分隔多段
- **不使用 notes**，所有信息通过标题直接展示
- 子节点用标签式分类：使用场景、关键辨析、注意事项、记忆技巧、常见错误、防范措施、代码示例
- 对比组间有 sizeof 差异、代码运行结果等量化对比节点

### 参考文件

- C语言指针辨析.xmind

---

## 通用规范

- 根节点统一使用 `org.xmind.ui.map.unbalanced` 布局
- 统一使用 XMind Zen（现代 JSON 格式）
- 技术术语保留英文原名，解释用中文
- 笔记中代码用反引号或缩进表示
- 可量化数据优先给出具体数值（如频率、容量、耗时）
